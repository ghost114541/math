# 从当前代码到最终成品的执行计划（一次性全功能版）

## Summary
当前项目已具备页面框架和交互雏形，但核心仍是“展示型原型”：2D/3D 画布未由真实矩阵链驱动、最终矩阵为模板、局部/世界坐标系未与变换同步。目标是一次性补齐为可教学交付的最终成品，并在 3D 中提供完整轨道控制与视角重置。

## Key Changes
1. 建立统一的仿射计算内核（2D/3D 共用）
- 新增纯函数计算层：参数解析 -> 单步矩阵生成 -> 矩阵链复合 -> 点/顶点批量变换。
- 支持 2D：平移/旋转/缩放（可保留 shear/reflect）；支持 3D：沿 xyz 平移、绕 xyz 旋转、xyz 缩放。
- 生成每一步矩阵 LaTeX 与最终矩阵 LaTeX，完全来自实时数据，不再硬编码。

2. 重构变换数据结构与状态流
- 将 `TransformItem.params: string` 升级为“结构化参数对象 + 可序列化显示文本”。
- 新增计算态：`basePoints`、`stepPoints[]`、`currentPoints`、`finalPoints`、`worldFrame`、`localFrameByStep[]`。
- 保持 UI 层只负责输入与展示，计算逻辑集中在 composable/engine，避免画布组件内重复算法。

3. 2D 画布改为真实逐步动画
- `P5CanvasStatic` 升级为动态渲染：按 `currentStep` 显示对应中间态，播放时执行插值过渡。
- 同时绘制原始图形、当前态、最终态（可分颜色/透明度），并支持顶点显示开关。
- 世界坐标系固定；局部坐标系按每步变换同步移动/旋转/缩放；`showWorld/showLocal` 与原点设置真实生效。

4. 3D 画布改为真实变换 + 完整相机交互
- `ThreeCanvasStatic` 升级为数据驱动：立方体顶点/网格按矩阵链更新，不再只做固定旋转演示。
- 增加完整轨道控制（旋转/缩放/平移）与“视角重置”按钮。
- 世界/局部坐标系可见性与步进同步，动画中当前步骤清晰可见。

5. 控制面板与输入系统补齐成品级可用性
- 点/顶点编辑支持增删改与齐次坐标约束（`w` 默认并锁定为 1，或提供受控编辑规则）。
- 变换参数输入改为按类型显示专用字段（不再依赖自由字符串）。
- 增加输入校验与错误提示（非法数字、缺失参数、动画播放时数据不完整）。
- 保留变换步骤增删与上下移动，矩阵高亮与动画步骤严格同步。

6. 教学说明与可扩展架构完善
- 每个示例补齐“主题、操作步骤、参数意义、观察点、理论映射”说明块。
- 将示例注册抽象成配置驱动（导航项 + 组件 + 元数据），新增示例只需注册配置。
- 页面信息结构固定化：标题区/说明区/参数矩阵区/画布区/动画区，便于后续扩展。

## Test Plan
1. 计算正确性测试（单元）
- 每类变换矩阵与期望值比对（2D/3D）。
- 复合顺序测试（`Tn ... T2 T1`）与点变换结果比对。
- 局部坐标系经同链变换后的结果校验。

2. 交互与状态测试（组件）
- 点编辑、增删点、变换增删/重排后，矩阵与画布同步刷新。
- 播放/暂停/单步/重置时，`currentStep`、矩阵高亮、画布中间态一致。
- `showWorld/showLocal`、局部原点修改后视觉结果一致。

3. 端到端场景（手工 + E2E）
- 2D：默认星形 -> 添加三步变换 -> 播放全过程 -> 校验最终图形与最终矩阵一致。
- 3D：默认立方体 -> 编辑顶点与旋转/缩放链 -> 轨道控制观察 -> 重置视角 -> 播放验证。
- 异常输入路径：空值/非法值/极值参数下 UI 提示与系统稳定性。

4. 交付验收标准
- 2D/3D 均可由用户输入驱动真实变换结果。
- 每步矩阵、最终矩阵、当前动画步骤高亮与画布同步。
- 世界/局部坐标系可控且在动画中同步更新。
- 3D 提供完整轨道控制与视角重置。
- `npm run build` 通过，关键交互无阻断性错误。

## Assumptions
- 采用“一次性全功能”路线，不拆分 MVP/增强版两期。
- 3D 相机交互按“完整轨道控制+重置”实现（教学演示优先）。
- 技术栈保持现有 Vue + p5 + three，不更换框架。
- 变换类型以课程核心为先：2D/3D 平移、旋转、缩放必须完成；shear/reflect 保留为可选增强但不影响主链验收。
